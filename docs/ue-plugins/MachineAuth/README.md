## 📜 3. 核心功能

### RSA 加密授权

- 基于 **RSA 2048 位加密**，安全可靠
- 采用 **PKCS#1 v1.5 格式**签名，与 C++ `RSA_public_decrypt` 兼容
- 授权码包含机器码、到期日期、最大使用次数等信息

### 多项目密钥管理

- 支持为**不同项目创建独立的密钥对**
- 每个项目使用不同的密钥对，互不干扰
- 即使某个项目的私钥泄露，其他项目不受影响

### 签发记录追踪

- 自动保存授权码签发历史
- 记录项目名称、机器码、授权天数、签发时间
- 方便查询和管理历史授权记录

## 🚀 4. 快速开始

### 第一步：创建密钥对

1. 打开在线授权工具（或 UE 插件界面）
2. 切换到「密钥管理」标签页
3. 点击「+ 新建密钥对」，输入项目名称和密钥长度
4. 生成后**导出私钥 .pem 文件**备份到安全位置
5. 复制公钥，粘贴到 UE5 代码的 `GetPublicKey()` 中

### 第二步：签发授权码

1. 在「生成授权码」页面选择密钥对
2. 输入客户机器码和授权天数
3. 点击生成，复制授权码发给客户

### 第三步：客户激活

1. 客户在 UE5 应用中粘贴授权码
2. `VerifyLicenseKey()` 用公钥解密并校验
3. 通过后 `SaveLicenseToLocal` 保存，下次自动验证

## 📖 5. 使用说明

### 关于密钥唯一性

每次生成 RSA 密钥对时，都会从两个超大素数中随机计算。  
2048 位密钥的碰撞概率为 1/2^2048（约 1/10^616），两个人生成相同密钥的概率为零。

### 推荐做法

**每个 UE5 项目使用独立的密钥对**，这样即使某个项目的私钥泄露，其他项目不受影响。

### 多项目架构示例

```
项目A 的 UE5 代码里 → 嵌入密钥对A 的公钥
项目B 的 UE5 代码里 → 嵌入密钥对B 的公钥
项目C 的 UE5 代码里 → 嵌入密钥对C 的公钥

用密钥对A签发的授权码 → 只能被项目A验证
用密钥对B签发的授权码 → 只能被项目B验证
```

**互不干扰，私钥泄露只影响单个项目** ✅

## 🛠️ 6. 常见问题 (FAQ)

**Q：密钥长度选择 1024、2048 还是 4096？**

A：推荐使用 **2048 位**，这是安全与速度的最佳平衡。
- 1024 位：较为简单，安全性较低
- 2048 位：推荐，安全且速度适中
- 4096 位：安全性最高，但生成较慢且授权码更长

**Q：私钥和公钥分别怎么使用？**

A：
- **私钥**：用于生成授权码，**绝不放入 UE 项目**，需妥善保管
- **公钥**：嵌入到 UE5 C++ 代码中，用于验证授权码

**Q：客户机器码怎么获取？**

A：在您的 UE 项目中调用 `GetMachineCode()` 函数获取客户机器的唯一标识码，发给您，您再用此工具生成授权码。
