关于密钥唯一性：每次生成 RSA 密钥对时，都会从两个超大素数中随机计算。 2048 位密钥的碰撞概率为 1/2^2048（约 1/10^616），两个人生成相同密钥的概率为零。

推荐做法：每个 UE5 项目使用独立的密钥对，这样即使某个项目的私钥泄露，其他项目不受影响。

完整使用流程

第一步：创建密钥对

  1. 切换到「密钥管理」标签页
  2. 点击「+ 新建密钥对」，输入项目名称
  3. 生成后导出私钥 .pem 文件备份到安全位置
  4. 复制公钥，粘贴到 UE5 代码的 GetPublicKey() 中

第二步：签发授权码

  1. 在「生成授权码」页面选择密钥对
  2. 输入客户机器码 and 授权天数
  3. 点击生成，复制授权码发给客户

第三步：客户激活

  1. 客户在 UE5 应用中粘贴授权码
  2. VerifyLicenseKey() 用公钥解密并校验
  3. 通过后 SaveLicenseToLocal 保存，下次自动验证

多项目架构

项目A 的 UE5 代码里 → 嵌入密钥对A 的公钥
项目B 的 UE5 代码里 → 嵌入密钥对B 的公钥
项目C 的 UE5 代码里 → 嵌入密钥对C 的公钥

用密钥对A签发的授权码 → 只能被项目A验证
用密钥对B签发的授权码 → 只能被项目B验证
互不干扰，私钥泄露只影响单个项目
