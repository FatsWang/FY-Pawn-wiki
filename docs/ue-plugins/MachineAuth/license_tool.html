推荐做法：每个 UE5 项目使用独立的密钥，这样即使某个项目的私钥泄露，其他项目不受影响。

完整使用流程

第一步：创建密钥

  1. 切换到「密钥管理」标签页
  2. 点击「+ 新建密钥」，输入项目名称
  3. 复制公钥，在 Create Widget 节点创建时传入 RSAPublicKey

第二步：签发授权码

  1. 在「生成授权码」页面选择密钥
  2. 输入客户机器码和授权天数
  3. 点击生成，复制授权码发给客户

第三步：客户激活

  1. 客户在 UE5 应用中粘贴授权码
  2. 点击授权界面中的 验证 按钮
  3. 插件校验机器码、有效期和使用次数
  4. 通过 OnMachineAuthResult 返回成功或失败结果

私钥安全

HTML 工具可以公开，公钥也可以放进 UE 项目；真正不能泄露的是私钥。 当前工具只把密钥保存在你选择的本地 JSON 文件中；浏览器最多保存文件句柄权限，不保存私钥内容。不要把私钥交给客户使用。

多项目架构

项目A 的 UE5 代码里 → 嵌入密钥A 的公钥
项目B 的 UE5 代码里 → 嵌入密钥B 的公钥
项目C 的 UE5 代码里 → 嵌入密钥C 的公钥

用密钥A签发的授权码 → 只能被项目A验证
用密钥B签发的授权码 → 只能被项目B验证
互不干扰，私钥泄露只影响单个项目
